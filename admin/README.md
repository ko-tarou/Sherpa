# Sherpa 管理者アプリ

全イベントの状態一覧と、論理削除チャンネル物理削除バッチの手動実行用UIです。

## セットアップ

1. `back/.env` に `ADMIN_API_KEY` を設定する。
2. バックエンドAPIを起動する（`cd back && make dev`）。
3. 管理者アプリの依存関係を入れ、起動する:

```bash
cd admin
npm install
npm run dev
```

4. ブラウザで http://localhost:5175 を開く。
5. ログイン画面で `ADMIN_API_KEY` の値を入力してログイン。

## 機能

- **全イベント一覧**: ID・タイトル・開始/終了・ステータス・スタッフ数・タスク（未着手/進行中/完了）・チャンネル数・作成日時を表で表示。
- **検索・フィルター**: イベント名・ID・場所で検索、ステータスでフィルター。
- **ソート**: 各列ヘッダークリックで昇順/降順切り替え。
- **バッチ実行**: 「バッチ実行」→ 確認 → 「実行」で、次を即時実行。
  - 論理削除済みチャンネル（およびメッセージ・メンバー）の物理削除
  - メンバー（EventStaff）が0人のイベント（および関連タスク・予算・チャンネル等）の物理削除

## 環境変数

| 変数 | 説明 |
|------|------|
| `VITE_API_URL` | バックエンドAPIのURL（未設定時は `http://localhost:3001`） |
